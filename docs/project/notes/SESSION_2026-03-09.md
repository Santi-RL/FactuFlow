## Problemas encontrados y como se resolvieron

### 1. Cache WSAA solo en memoria

Problema:
- el TA válido no quedaba reutilizable entre procesos

Solucion:
- persistir tickets en `backend/data/arca_token_cache.json`
- archivo tocado: `backend/app/arca/cache.py`

### 2. `FEParamGetPtosVenta` devolvia `602`

Problema:
- homologación respondía `Sin Resultados` aunque el punto de venta sí servía

Solucion:
- tolerar `602` solo en homologación
- validar con `FECompUltimoAutorizado` y emisión real
- archivos: `backend/app/arca/wsfev1.py`, `backend/app/services/facturacion_service.py`

### 3. Estructura SOAP incorrecta en `FECAESolicitar`

Problema:
- errores al construir `FeDetReq`, `Iva` y `Tributos`

Solucion:
- ajustar la estructura enviada al servicio
- archivo: `backend/app/arca/wsfev1.py`

### 4. Faltaba `CondicionIVAReceptorId`

Problema:
- ARCA rechazo con código `10246`

Solucion:
- mapear la condicion IVA del receptor y enviarla en el request
- archivos: `backend/app/arca/models.py`, `backend/app/arca/wsfev1.py`, `backend/app/services/facturacion_service.py`

### 5. Detalle de comprobante fallaba con error 500

Problema:
- SQLAlchemy exigia `unique()` al cargar ítems con `joinedload`

Solucion:
- usar `result.unique().scalar_one_or_none()`
- archivo: `backend/app/api/comprobantes.py`
- test agregado: `backend/tests/test_comprobantes_api.py`

### 6. `Ver PDF` devolvia 404

Problema:
- el frontend llamaba `/pdf/...` pero el backend exponia `/api/pdf/...`

Solucion:
- corregir rutas en `frontend/src/services/pdf.service.ts`

### 7. DB local legacy fuera de Alembic

Problema:
- la base local existente no estaba alineada con la historia de migraciones

Solucion aplicada solo para destrabar smoke:
- crear tablas faltantes de lotes sobre la base local existente

Pendiente real:
- definir estrategia limpia de stamping/migración para bases legacy
